# МедГрафик — Система управления расписанием клиники

Веб-приложение для управления расписанием врачей, записью пациентов и аналитикой утилизации.

## Технологии

- **React 18** — UI-фреймворк
- **Vite** — сборщик
- **Tailwind CSS** — стили
- **dayjs** — работа с датами (с русской локалью)
- **Recharts** — графики
- **React Router v6** — маршрутизация
- **uuid** — генерация уникальных ID

## Как запустить

```bash
npm install
npm run dev
```

Приложение откроется на `http://localhost:5173`

## Тестовые учётные данные

| Роль | Email | Пароль | Возможности |
|------|-------|--------|-------------|
| Администратор | `admin@med.ru` | `admin` | Все функции |
| Врач | `doc@med.ru` | `doc` | Просмотр расписания |
| Регистратор | `reg@med.ru` | `reg` | Запись пациентов |

## Страницы приложения

### 📊 Дашборд
- Карточки с KPI: количество слотов, записей, утилизация, отмены
- График утилизации по врачам за последние 7 дней
- Рейтинг врачей по загрузке

### 📅 Расписание врача
- Просмотр расписания в режиме «День» и «Неделя»
- Цветовая индикация слотов: свободный / занят / завершён / отменён
- Список записанных пациентов на день

### 📋 Управление записями *(только регистратор)*
- Пошаговая запись пациента: выбор врача → дата → слот → данные пациента
- Список всех активных записей
- Отмена записи с указанием причины
- Перенос записи на другое время

### 👨‍⚕️ Управление врачами *(только администратор)*
- Список всех врачей с информацией о кабинете
- Создание расписания: рабочие часы, длительность слота, перерыв
- Просмотр существующих расписаний с возможностью удаления

### 📈 Отчёты и аналитика
- Анализ утилизации по врачам и по датам
- Фильтрация по периоду и специальности
- Экспорт в CSV (с поддержкой кириллицы в Excel)

## Роли и права доступа

## Что было исправлено

### Исправления из первого PR

1. **Убраны отладочные `console.log`** в `Reports.jsx`  
   В функции `extractLastName` и после вычисления данных графика были оставлены отладочные вызовы, которые выводили данные в консоль браузера в продакшн-режиме.

2. **Исправлен экспорт CSV с кириллицей**  
   Файл CSV при открытии в Microsoft Excel показывал кракозябры вместо русских букв. Добавлен BOM-маркер (`\uFEFF`) в начало файла — теперь Excel правильно определяет кодировку UTF-8.

3. **Исправлена автоматическая привязка врача**  
   Страница «Расписание врача» должна автоматически открывать расписание залогиненного врача, но у тестового пользователя `doc@med.ru` не было поля `doctorId`. Добавлено `doctorId: "doc-1"` в объект пользователя-врача в `AuthContext.jsx`.

### Исправления из второго PR

4. **Добавлено удаление расписания**  
   В предупреждении на странице «Управление врачами» упоминалось: «удалите существующее расписание» — но кнопки удаления не было. Добавлены: экшн `DELETE_SCHEDULE` в `ScheduleContext`, функция `deleteSchedule`, кнопка удаления в карточке расписания.

5. **Дата записи по умолчанию — завтра**  
   На странице записи пациента дата по умолчанию была сегодняшняя, хотя слоты обычно создаются на будущие даты. Изменено на «завтра».

6. **Убраны неиспользуемые переменные и комментарии**  
   - Удалены переменные `endsAfterBreak` и `startsBeforeBreak` в `scheduleUtils.js` (объявлялись, но не использовались)  
   - Убраны отладочные маркеры `// 🔴` из `Dashboard.jsx` и `scheduleUtils.js`

## Что ещё можно улучшить

### Высокий приоритет

- **Сохранение данных** — сейчас все данные хранятся в памяти и теряются при перезагрузке страницы. Нужно подключить localStorage или бэкенд-API.
- **Добавление новых врачей** — в системе нет UI для создания нового врача, только просмотр существующих.
- **Реальный бэкенд** — все данные сейчас захардкожены в `mockData.js`. Нужна база данных и REST/GraphQL API.

### Средний приоритет

- **Проверка дублей пациентов** — можно записать одного пациента дважды на разное время к одному врачу в один день.
- **Статус «Врач недоступен»** — нет возможности отметить, что врач в отпуске или болеет.
- **Поиск пациентов** — нет справочника пациентов, каждый раз нужно вводить ФИО вручную.
- **Уведомления** — нет напоминаний пациентам о записи (email / SMS).

### Низкий приоритет

- **Тёмная тема**
- **Мобильное приложение**
- **Права доступа на уровне отдельных действий** (сейчас только по роли)
- **Журнал операций** — данные пишутся в `operationLog`, но нет страницы для их просмотра
